# Cloud Album

本仓库使用根目录 Markdown 文档保存可恢复的项目状态。重要状态写入文件，长线程或上下文压缩后从文件恢复，而不是依赖对话记忆。

## 文档职责

- `OVERVIEW.md` - 项目整体功能描述。
- `PLAN.md` - 任务状态和验证标准。
- `STRUCTURE.md` - 架构参考。
- `MEMORY.md` - 发现、特殊情况、变通方案，以及有效或失败的做法。

## 流程

```text
用户请求
    |
    +- 检查 PLAN.md 是否存在
    |   +- 如果存在：读取 PLAN.md、STRUCTURE.md、MEMORY.md，再进入任务执行
    |   +- 如果不存在：读取 OVERVIEW.md，继续新的规划流程
    |
    +- 分析风险并定义验证标准 -> PLAN.md
    +- 设计或更新架构 -> STRUCTURE.md
    +- 向用户简要说明计划摘要
    +- 执行任务
    +- 完成后更新 PLAN.md，写发现到 MEMORY.md
```

## OVERVIEW.md

`OVERVIEW.md` 保存项目整体功能描述。

## PLAN.md

`PLAN.md` 保存任务状态和验证标准。

格式：

````markdown
# 项目计划：{名称}

## 风险任务

{如果没有风险任务，整节省略。}

### 1. {风险功能}
- **隔离原因：** {为什么这个部分有独立验证风险}
- **做法：** {关键策略或约束，足够让实现者知道怎么做}
- **验证：** {针对失败模式的具体验证标准}

## 主要构建

{要构建的常规系统。高层描述，不写实现细节。}

- **所需资源：** {需要的资源：类型、尺寸、用途。没有则省略。}
- **验证：**
  - {通用检查}
  - {项目特定检查}
  - 用户流程符合需求描述
  - 界面可读，没有溢出或重叠
  - 没有缺失资源、占位内容或明显回退状态
````

## STRUCTURE.md

`STRUCTURE.md` 是完整架构参考。每次更新都写成完整版本，不写差异。

格式：

````markdown
# {项目名称}

## 架构

{完整架构参考。}
````

## MEMORY.md

`MEMORY.md` 保存发现、特殊情况、变通方案，以及有效或失败的做法。

完成每个任务后：更新 `PLAN.md`，写发现到 `MEMORY.md`。如果线程变得嘈杂，把重要状态总结到这些文件中，然后从文件继续，而不是依赖对话记忆。
