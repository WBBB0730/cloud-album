## 风险任务

### 1. 私有 COS 上传、读取和权限隔离
- **隔离原因：** 家庭相册的隐私要求高，上传使用浏览器直传 COS，读取使用短期签名 URL；如果空间校验或签名范围处理错误，可能导致跨空间访问或文件外泄。
- **做法：** 所有 Server Action 在签发上传/读取授权前校验当前用户属于目标空间；COS bucket 使用私有读写；客户端只获得指定 object key 的最小权限；数据库记录必须带 `space_id`。
- **验证：** 已在服务层实现空间成员校验；构建通过。仍需在真实数据库和 COS bucket 上验证非成员访问、签名 URL 过期、跨空间传参拒绝。

### 2. 大文件和视频分片上传
- **隔离原因：** 视频体积不可控，不能依赖普通表单上传或 Server Action 中转；失败重试和续传会影响上传状态一致性。
- **做法：** 浏览器直传 COS；图片可走普通直传，视频和大文件走分片上传；使用 `upload_sessions` 记录上传状态；上传完成后再确认媒体可见。
- **验证：** 已使用 COS JS SDK `sliceUploadFile` 分片上传和上传队列；失败媒体保持 `failed` 不进普通列表。仍需真实 COS CORS 配置后验证大视频、失败重试和续传表现。

### 3. 逻辑删除、永久删除和恢复批次
- **隔离原因：** 文件夹删除会联动媒体，恢复时需要准确恢复同一批次内容；永久删除又不能真正删除 COS 文件。
- **做法：** 文件夹和媒体都记录 `deleted_at`、`deleted_by`、`permanently_deleted_at`、`permanently_deleted_by`、`delete_batch_id`；删除文件夹时给文件夹和联动媒体写入同一个批次。
- **验证：** 已实现普通删除、回收站、恢复、永久删除和后台恢复。仍需数据库迁移后进行端到端数据验证。

### 4. 拍摄时间解析与排序
- **隔离原因：** 图片 EXIF 和视频元数据不一定存在或可读，但第一版默认排序依赖拍摄时间。
- **做法：** 上传确认阶段尽量解析图片 EXIF 和视频元数据；解析失败时写入上传时间作为 `taken_at`；所有媒体列表默认按 `taken_at` 排序。
- **验证：** 已在上传客户端尽力读取图片 EXIF、视频 metadata，失败回退上传时间；媒体列表按 `taken_at` 倒序。仍需真实文件样本验证。

## 主要构建

第一版构建一个私有家庭云相册，包含邀请注册、手机号密码登录、空间、一级文件夹、图片/视频上传与预览、回收站、全局管理后台。

- **所需资源：**
  - PostgreSQL 数据库。
  - 腾讯云 COS 私有 bucket。
  - COS 临时授权或服务端签名配置。
  - 全局管理员初始化配置。
- **验证：**
  - 注册邀请只能由全局管理员创建和撤销。
  - 用户可以通过绑定手机号的邀请链接设置密码注册。
  - 用户可以创建空间并加入多个空间。
  - 用户默认进入空间列表。
  - 空间数据隔离正确。
  - 用户可以创建一级文件夹。
  - 用户可以多文件上传图片和视频。
  - 视频和大文件支持分片上传、失败重试和续传。
  - 图片和视频可以浏览、预览、删除、恢复。
  - 回收站首页按文件夹展示，进入文件夹后展示已删除媒体。
  - 删除和永久删除都不删除 COS 文件。
  - 用户流程符合需求描述。
  - 界面可读，没有溢出或重叠。
  - 没有缺失资源、占位内容或明显回退状态。
