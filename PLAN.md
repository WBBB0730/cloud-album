## 持续风险

- [ ] 私有 COS 上传、读取和权限隔离。
  - 签发上传/读取授权前必须校验空间成员。
  - COS object key 禁止前导 `/`。
  - 上传意图创建时携带 `contentHash`；同相册已有 ready 媒体命中相同 hash 时直接视为处理完成，不再签发 COS 上传凭证。
  - 分享导入最终仍必须走原上传页的上传意图和 COS 直传流程，不能让 Next 服务端接收并暂存媒体正文，也不能在导入页另起一套上传完成逻辑或改变原上传组件内部上传策略。
  - 真实环境验证非成员访问拒绝、稳定媒体路由协商缓存 304/Range/权限失效、跨空间传参拒绝、大文件分片上传。
- [ ] 媒体密集页面性能。
  - 相册缩略图保持懒加载。
  - 相册网格和封面必须使用 `thumbnailUrl`，不能直接显示原图 URL。
  - 页面数据缓存需要有版本或迁移/校验逻辑；媒体列表字段契约变化时，旧缓存不能直接进入渲染。
  - `thumbnailUrl` 指向 `/api/media/[mediaId]/preview`，图片缩略图和视频封面由数据万象 CI 运行时生成。
  - 预览主区域只挂载当前项和前后 2 项原图/原视频。
  - 刷新数据时保持媒体对象和可用 URL 稳定。
  - 预览大图保持原图，不恢复已验证有副作用的延迟替换方案。
  - 真实数据验证首次打开、快速滚动、预览连续切换的卡顿和掉帧。
- [ ] 删除、恢复和封面一致性。
  - 批量删除使用事务和批量更新。
  - 删除包含当前封面时，同一事务内切换到最新可用媒体或清空封面。
  - 普通删除和永久删除都不删除 COS 文件。
  - 复制媒体会让多个媒体记录共享同一个 COS key；删除和永久删除仍然只能改数据库状态，不能删除 COS 对象。
  - 相册间复制前的目标重复检查必须走相册级部分索引，避免大相册或复用 COS key 后退化为全表过滤。
- [ ] 客户端壳和移动端导航手势。
  - 页面只渲染客户端壳，业务数据通过 Server Action 拉取。
  - 固定返回、多选返回、预览返回、上传离开保护和下拉刷新不能互相干扰。
  - 物理返回验证需要从新标签页输入 `localhost:3000` 开始，使用浏览器顶部返回按钮验证 `/spaces -> 空间 -> 相册 -> 空间 -> /spaces -> 新标签页`。
  - 选择态物理返回验证需要覆盖：进入相册后长按媒体进入选择态，第一次顶部返回只取消选择，第二次回空间页，第三次回选择空间页。
  - 页面内返回和物理返回混用也必须验证：进入相册后点击页面内返回到空间页，再按顶部物理返回，必须到选择空间页，不能回到相册。
- [ ] Android PWA 分享入口。
  - Mac 只能模拟 Web Share Target 的 service worker/IndexedDB/导入页链路；系统相册分享面板、WebAPK 安装刷新和 MIME 匹配必须用 Android 真机最终验收。
  - manifest 或 service worker 变更后，测试机可能需要重新安装 PWA 才能刷新系统分享目标。
  - 分享导入临时批次保存在 IndexedDB；原上传页成功读取并排入普通上传队列后清理该批次，超过 TTL 的批次由后续入口清理，避免长期占用浏览器存储。

## 每次改动检查

- [ ] 遵守用户当轮需求，不擅自扩大范围。
- [ ] 如果看到文件内容和上次修改后不一致，优先视为用户手动修改。
- [ ] 涉及 UI 时同步 `docs/design/index.html`。
- [ ] 涉及 UI 时检查移动端真实视口、安全区域、滚动区域和按钮对齐。
- [ ] 不新增 `vh`、`svh`、`h-screen`、`min-h-screen` 等传统视口高度写法。
- [ ] 根据改动范围运行 `pnpm.cmd typecheck`。
- [ ] 涉及构建、路由模式或 PWA 配置时运行 `pnpm.cmd build`。
- [ ] 涉及数据库结构时生成并验证 migration。
- [ ] 涉及媒体、上传、删除、权限时用真实数据库和真实 COS 行为验证。
