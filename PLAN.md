## 每次改动检查

- [ ] 遵守用户当轮需求，不擅自扩大范围。
- [ ] 如果看到文件内容和上次修改后不一致，优先视为用户手动修改。
- [ ] 涉及 UI 时同步 `docs/design/index.html`。
- [ ] 涉及 UI 时检查移动端真实视口、安全区域、滚动区域和按钮对齐。
- [ ] 不新增 `vh`、`svh`、`h-screen`、`min-h-screen` 等传统视口高度写法。
- [ ] 根据改动范围运行 `pnpm.cmd typecheck`。
- [ ] 涉及构建、路由模式或 PWA 配置时运行 `pnpm.cmd build`。
- [ ] 涉及数据库结构时生成并验证 migration。
- [ ] 涉及媒体、上传、删除、权限时用真实数据库和真实 COS 行为验证。
